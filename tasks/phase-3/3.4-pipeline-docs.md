## Problem

Developers must read a 782-line registrar to understand what happens to a message between arriving from Redis and reaching their handler. There is no high-level explanation anywhere.

Add a new section **"Message Processing Pipeline"** between the annotations reference section and the end-to-end flows section.

```markdown
## Message Processing Pipeline

Every `@RacerListener` and `@RacerStreamListener` method processes messages through
the same pipeline, in this exact order:

  ┌─────────────┐     ┌───────────────┐     ┌─────────────────┐
  │  Receive     │ --> │  Parse JSON   │ --> │  Schema         │
  │  (Pub/Sub or │     │  into         │     │  Validation     │
  │   XREADGROUP)│     │  RacerMessage │     │  (if enabled)   │
  └──────────────┘     └───────────────┘     └────────┬────────┘
                                                       │ fail → DLQ
                                                       ▼
  ┌─────────────┐     ┌───────────────┐     ┌─────────────────┐
  │  Circuit     │ <-- │  Dedup Check  │ <-- │  Content        │
  │  Breaker     │     │  (if enabled) │     │  Routing        │
  │  Gate        │     │               │     │  (if configured)│
  └──────┬───────┘     └───────────────┘     └─────────────────┘
  open → DLQ
         │
         ▼
  ┌─────────────┐     ┌───────────────┐     ┌─────────────────┐
  │  Interceptor │ --> │  Argument     │ --> │  YOUR HANDLER   │
  │  Chain       │     │  Resolution   │     │  METHOD         │
  └──────────────┘     └───────────────┘     └────────┬────────┘
                                                       │
                                              success  │  failure
                                                ▼            ▼
                                          ┌──────────┐  ┌─────────┐
                                          │ Metrics  │  │  DLQ    │
                                          │ +1       │  │ enqueue │
                                          └──────────┘  └─────────┘

### Stage details

| Stage | Always Active? | Opt-in property |
|-------|---------------|-----------------|
| Receive (Pub/Sub or XREADGROUP) | Yes | — |
| Parse JSON into `RacerMessage` | Yes | — |
| Schema Validation | No | `racer.schema.enabled=true` |
| Content Routing | No | `@RacerRouter` DSL or `@RouteTo` |
| Dedup Check | No | `racer.dedup.enabled=true` |
| Circuit Breaker Gate | No | `racer.circuit-breaker.enabled=true` |
| Interceptor Chain | No | Implements `RacerListenerInterceptor` |
| Argument Resolution | Yes | — |
| Metrics | No | Micrometer on classpath |
| DLQ | No | `racer.dlq.enabled=true` |
```
