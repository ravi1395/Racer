# Task 4.3 — Consolidate Constructor Overloads

**Phase:** v2.0.0  
**Risk:** Medium  
**Effort:** ~2 days  
**Breaking:** **Yes** — extra constructors removed

## Problem

`RacerPublisherRegistry` has 5 constructors and `RacerCircuitBreakerRegistry` has 2 constructors accumulated for backward compatibility as features were added incrementally. In a major version, these should be consolidated.

## Files to Change

### 1. `publisher/RacerPublisherRegistry.java`

**Remove** the 4 shorter constructors (the ones that omit metrics, schema validation, or rate limiting).

**Keep** only the full constructor that accepts all dependencies.

**Add** a `forTesting()` static factory for test convenience:

```java
/**
 * Creates a minimal registry for testing — no metrics, no schema validation,
 * no rate limiting. Passes {@link Optional#empty()} for all optional dependencies.
 *
 * <p>Usage:</p>
 * <pre>
 * RacerPublisherRegistry registry = RacerPublisherRegistry.forTesting(props, template, mapper);
 * </pre>
 */
public static RacerPublisherRegistry forTesting(
        RacerProperties props,
        ReactiveRedisTemplate<String, String> template,
        ObjectMapper mapper) {
    return new RacerPublisherRegistry(props, template, mapper,
            Optional.empty(),   // metrics
            Optional.empty(),   // schemaRegistry
            Optional.empty());  // rateLimiter
}
```

### 2. `circuitbreaker/RacerCircuitBreakerRegistry.java`

**Remove** the no-metrics constructor. Any caller must now pass `new NoOpRacerMetrics()` (or a real `RacerMetrics` instance) explicitly.

Before removing: grep for call sites:

```bash
grep -r "new RacerCircuitBreakerRegistry(" src/
```

Update each call site that used the no-metrics constructor to pass `new NoOpRacerMetrics()`.

Expected locations:

## Migration for Test Code

Before v2.0.0, tests may use:
```java
new RacerPublisherRegistry(props, template, mapper)
```

After v2.0.0, they must use:
```java
RacerPublisherRegistry.forTesting(props, template, mapper)
```
