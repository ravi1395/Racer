# Task 4.2 — Remove Deprecated Router Sync Path

**Phase:** v2.0.0  
**Risk:** Medium  
**Effort:** ~2 days  
**Breaking:** **Yes** — synchronous `route()` and `evaluate()` methods removed

## Problem

`RacerRouterService.route(RacerMessage)` and `evaluate(RacerMessage, List<CompiledRouteRule>)` are synchronous, deprecated methods that block the Reactor thread. The reactive variants `routeReactive()` and `evaluateReactive()` were added as replacements but the sync methods were never removed, leaving two parallel APIs.

## Files to Change

### 1. `router/RacerRouterService.java`

**Step 1 — Remove the sync methods:**
- Delete `route(RacerMessage msg)` (synchronous)
- Delete `evaluate(RacerMessage msg, List<CompiledRouteRule> rules)` (synchronous)

**Step 2 — Rename the reactive methods to claim the canonical names:**
- Rename `routeReactive(RacerMessage msg)` → `route(RacerMessage msg)`
- Rename `evaluateReactive(RacerMessage msg, List<CompiledRouteRule> rules)` → `evaluate(RacerMessage msg, List<CompiledRouteRule> rules)`

After renaming, the API is clean: `route()` and `evaluate()` are the only methods, and they are reactive.

### 2. `listener/RacerListenerRegistrar.java`

Search for any call sites using the old method names (`routeReactive`, `evaluateReactive`) and update them to the new names (`route`, `evaluate`).

```java
// BEFORE (inside message processing pipeline):
router.routeReactive(msg).flatMap(decision -> ...)

// AFTER:
router.route(msg).flatMap(decision -> ...)
```

## Grep for Call Sites

Before making changes, search the codebase for all usages:

```bash
grep -r "routeReactive\|evaluateReactive" src/
```

Expected hits: `RacerListenerRegistrar.java` (and possibly `RacerStreamListenerRegistrar.java`).

## Tests to Verify

No new tests needed for the removal itself. Verify by confirming that:

1. The existing integration test `PubSubIntegrationTest` (which exercises routing) still passes
2. The existing `RacerRouterServiceTest` still passes with the renamed methods

If any test class calls `routeReactive()` or `evaluateReactive()` directly, update those calls.
