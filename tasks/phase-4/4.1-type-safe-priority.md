# Task 4.1 — Type-Safe Priority in `RacerMessage`

**Phase:** v2.0.0  
**Risk:** High  
**Effort:** ~4–5 days  
**Breaking:** **Yes** — `getPriority()` return type changes from `String` to `PriorityLevel`

## Problem

`RacerMessage.priority` is a raw `String`. Typos like `"HGIH"` compile and silently create a value that the priority publisher doesn't recognize, causing messages to be misrouted at runtime with no compile-time error.

## Files to Change

### 1. `model/PriorityLevel.java` — Add `fromString()` (if missing)

Verify the enum has a `fromString` method for Jackson deserialization backward compatibility. If absent, add:

```java
/**
 * Parses a priority level from its string name (case-insensitive).
 * Returns {@link #NORMAL} for unrecognized values and logs a warning
 * to maintain backward compatibility with legacy JSON payloads.
 */
public static PriorityLevel fromString(String name) {
    try {
        return PriorityLevel.valueOf(name.toUpperCase());
    } catch (IllegalArgumentException ex) {
        log.warn("[racer] Unknown priority '{}' — defaulting to NORMAL", name);
        return NORMAL;
    }
}
```

### 2. `model/RacerMessage.java` — Change field type

```java
// BEFORE:
@Builder.Default
private String priority = "NORMAL";

// AFTER:
/**
 * Priority level for priority-channel routing.
 * Uses the {@link PriorityLevel} enum for type safety.
 * Jackson deserializes legacy {@code "HIGH"} string values via
 * {@link PriorityLevel#fromString(String)}.
 */
@Builder.Default
private PriorityLevel priority = PriorityLevel.NORMAL;
```

Also add a Jackson custom deserializer or `@JsonCreator` on `PriorityLevel.fromString` so existing JSON with `"priority": "HIGH"` still deserializes correctly:

```java
// On PriorityLevel enum:
@JsonCreator
public static PriorityLevel fromString(String name) { ... }
```

### 3. `publisher/RacerPriorityPublisher.java`

Update any method signatures that accept `String priority` to `PriorityLevel priority`.

### 4. `publisher/MessageEnvelopeBuilder.java`

Ensure `PriorityLevel` is serialized as its string name on the wire:

```java
// Jackson serializes enums by name by default — confirm no custom serializer conflicts
// If explicit, use:
envelope.put("priority", message.getPriority().name());
```

### 5. `annotation/PublishResult.java`

```java
// BEFORE:
String priority() default "NORMAL";

// AFTER:
PriorityLevel priority() default PriorityLevel.NORMAL;
```

### 6. `annotation/RacerPriority.java`

```java
// BEFORE:
String defaultLevel() default "NORMAL";

// AFTER:
PriorityLevel defaultLevel() default PriorityLevel.NORMAL;
```

### 7. `aspect/PublishResultAspect.java`

Where `@PublishResult(priority="HIGH")` was read as a String, it is now already a `PriorityLevel`. Remove any `PriorityLevel.fromString()` call that was added as a workaround, or keep it if the annotation still uses String (deprecated path).

### 8. `listener/RacerListenerRegistrar.java`

Update priority handling in the processing pipeline wherever `msg.getPriority()` is compared to String literals.

## Wire Compatibility

**On the Redis wire, priority must remain a plain string** for backward compatibility with older publishers that serialize `"priority": "HIGH"`. Jackson's default enum serialization matches this behavior (writes the enum name). No wire change required.
