# Task 4.4 — Remove Deprecated `dryRun(Object)`

**Phase:** v2.0.0  
**Risk:** Low  
**Effort:** ~0.5 days  
**Breaking:** **Yes** — deprecated method removed  
**Prerequisite:** Task [3.5](../phase-3/3.5-router-dryrun-fix.md) must be complete (it introduces the three-arg `dryRun` and deprecates the one-arg form)

## Problem

`RacerRouterService.dryRun(Object payload)` was deprecated in v1.6.0 (task 3.5) because it silently skips SENDER and ID rules. The v2.0.0 cleanup removes it entirely.

## File to Change

### `router/RacerRouterService.java`

**Delete** the single-argument `dryRun(Object payload)` method entirely:

```java
// DELETE this method (was deprecated since 1.6.0):
@Deprecated(since = "1.6.0", forRemoval = true)
public RouteDecision dryRun(Object payload) {
    return dryRun(payload, null, null);
}
```

The three-argument method introduced in 3.5 becomes the only `dryRun` API:

```java
// This remains:
public RouteDecision dryRun(Object payload, @Nullable String sender, @Nullable String messageId) {
    ...
}
```

## Call Site Audit

Before deleting, confirm no internal code calls the one-arg form:

```bash
grep -rn "\.dryRun(" src/
```

If any internal call sites exist, update them to `dryRun(payload, null, null)`.

## Tests to Update

If `RacerRouterServiceDryRunTest` (written in 3.5) has a sub-test that calls `dryRun(payload)`, remove that sub-test or update it to use `dryRun(payload, null, null)`.

## Acceptance Criteria

- [x] `dryRun(Object)` method deleted from `RacerRouterService`
- [x] `dryRun(Object, String, String)` retained as the sole dry-run API
- [x] No compilation errors anywhere in the codebase
